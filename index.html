Les 3 outils que tu vas utiliser

Tu n'as pas besoin d'apprendre à coder. Ces 3 outils font le travail à ta place.

🤖

Claude Code

L'assistant IA qui écrit le code pour toi. Tu lui parles en français. Il fait ce que tu lui demandes.

🗂️

Git & GitHub

L'outil qui sauvegarde ton travail. Comme l'historique de versions dans Google Docs, mais pour ton code.

🚀

Vercel

Le service qui met ton site en ligne automatiquement. Tu pousses ton code, le site se met à jour tout seul.

1. Claude Code — l'assistant IA

Claude Code, c'est quoi ?
C'est un assistant IA qui travaille directement dans ton ordinateur. Tu lui décris ce que tu veux faire. Il écrit le code pour toi. Tu peux lui parler en français, comme si c'était un collègue.

À retenir

Claude Code n'est pas ChatGPT. Il a accès à tes fichiers. Il peut modifier ton code directement. Il faut lui donner des instructions précises.

Comment parler à Claude Code

La méthode la plus simple : décris ce que tu veux en 3 points.

  • 1 Ce que tu veux faire. Exemple : "Je veux créer une page d'accueil pour mon application de gestion de rendez-vous."
  • 2 Le style à respecter. Exemple : "La couleur principale est le bleu #1A73E8. La police est Inter. Il faut que ce soit professionnel."
  • 3 Les règles importantes. Exemple : "Respecte l'accessibilité web. Pas de cookies sans consentement."

Exemple de message à envoyer

Exemple de prompt

"Crée une page d'accueil pour mon outil de gestion de projets. La couleur principale est le violet #6B21A8. Le titre est : Planoria, gère tes projets simplement. Ajoute un bouton "Commencer" bien visible. La page doit être accessible : texte lisible, contrastes suffisants, navigation au clavier."

Conseil pratique

Si le résultat ne te convient pas, dis-le à Claude Code. Exemple : "Le bouton est trop petit. Agrandis-le." Tu peux corriger autant de fois que tu veux.

2. Git et GitHub — sauvegarder ton travail

Git, c'est l'outil qui sauvegarde chaque modification de ton code. À chaque étape importante, tu crées un "point de sauvegarde". Si tu fais une erreur, tu peux revenir en arrière.

GitHub, c'est le site internet où ton code est stocké en ligne. Toute ton équipe peut y accéder. Vercel surveille GitHub pour mettre ton site à jour.

Analogie

Git, c'est comme l'historique de versions de Google Docs. GitHub, c'est comme Google Drive : tes fichiers sont en ligne, accessibles depuis n'importe où.

Les 3 actions que tu feras tout le temps

  1. 1
    Préparer la sauvegarde

    Tu sélectionnes les fichiers que tu veux sauvegarder. En langage Claude Code : "Prépare tous mes fichiers modifiés pour la sauvegarde."

  2. 2
    Créer le point de sauvegarde

    Tu enregistres tes modifications avec un message qui explique ce que tu as fait. Exemple de message : "Ajout de la page contact".

  3. 3
    Envoyer sur GitHub

    Tu envoies tes sauvegardes sur le site GitHub. Dès que c'est fait, Vercel met ton site à jour automatiquement.

Ce que tu peux dire à Claude Code

"Sauvegarde mon travail sur GitHub. Le message de sauvegarde est : Ajout du formulaire de contact."

Claude Code gère toutes les commandes à ta place.

Créer ton premier dépôt GitHub

Un dépôt (en anglais : repository), c'est le dossier en ligne de ton projet.

  1. 1
    Crée un compte sur github.com

    C'est gratuit. Utilise ton adresse email professionnelle.

  2. 2
    Crée un nouveau dépôt

    Clique sur "New repository". Donne un nom à ton projet. Coche "Private" si ton projet ne doit pas être public.

  3. 3
    Connecte Claude Code à GitHub

    Dis à Claude Code : "Connecte ce projet à mon dépôt GitHub [lien de ton dépôt]."

3. Vercel — mettre ton site en ligne

Vercel est le service qui publie ton site sur internet. Tu n'as rien à faire manuellement. Quand tu envoies ton code sur GitHub, Vercel le détecte et met le site à jour.

Vercel donne à ton site une adresse en ligne. Exemple : mon-projet.vercel.app. Tu peux aussi brancher ton propre nom de domaine.

Bon à savoir

Vercel est gratuit pour les petits projets. Pour un usage professionnel avec plus de trafic, il existe des offres payantes.

Mettre ton projet en ligne en 4 étapes

  1. 1
    Crée un compte sur vercel.com

    Clique sur "Sign up". Choisis "Continue with GitHub". Vercel se connecte à ton compte GitHub.

  2. 2
    Importe ton projet GitHub

    Dans Vercel, clique sur "Add New Project". Choisis ton dépôt GitHub dans la liste.

  3. 3
    Lance le déploiement

    Clique sur "Deploy". Vercel met ton site en ligne. Tu reçois une adresse web.

  4. 4
    Ensuite, c'est automatique

    À chaque fois que tu envoies du nouveau code sur GitHub, Vercel met le site à jour. Tu n'as rien d'autre à faire.

À retenir

Le flux complet est :
Claude Code crée le code → GitHub le sauvegarde → Vercel le met en ligne.

4. La charte graphique — le style de ton produit

La charte graphique, c'est le "style" de ton produit. Elle définit les couleurs, les polices de caractères, et l'aspect visuel général. Grâce à elle, ton produit a l'air professionnel et cohérent partout.

Quand tu travailles avec Claude Code, tu lui fournis la charte graphique dès le début. Il la respecte dans tout le code qu'il crée.

Les éléments d'une charte graphique simple

  • 🎨 Les couleurs. Choisis 2 ou 3 couleurs maximum. Une couleur principale, une couleur secondaire, une couleur de texte. Exemple : bleu #1A73E8, blanc #FFFFFF, texte gris foncé #1A1A1A.
  • 🔤 La police de caractères. Choisis une seule police pour commencer. Exemple : Inter, Roboto, ou Arial. Ces polices sont gratuites et lisibles.
  • 📐 Les formes. Décide si les boutons sont arrondis ou droits. Si les cadres ont des coins ronds ou non. La cohérence compte plus que la perfection.
  • 🖼️ Le logo ou l'icône. Si tu as un logo, fournis-le à Claude Code. Il l'intégrera dans les pages.

Comment créer ta charte avec Claude Code

  1. 1
    Décris ton univers visuel

    Dis à Claude Code : "Mon produit s'appelle [nom]. C'est un outil pour [public cible]. Je veux un style [moderne / rassurant / dynamique / professionnel]."

  2. 2
    Donne tes couleurs

    Si tu as déjà des couleurs, donne les codes hexadécimaux (ex: #FFF44F). Sinon, demande à Claude Code de proposer une palette.

  3. 3
    Demande un fichier de charte

    Dis-lui : "Crée un fichier de charte graphique pour mon projet avec toutes ces décisions de design."

  4. 4
    Répète la charte dans chaque instruction

    À chaque nouvelle page, rappelle à Claude Code : "Respecte la charte graphique du fichier [nom du fichier]."

Important : les contrastes de couleurs

Les couleurs de ta charte doivent avoir un contraste suffisant. Texte clair sur fond clair = illisible. Texte foncé sur fond clair = lisible. C'est aussi une obligation légale d'accessibilité (voir section 6). Tu peux tester sur le site contrast-ratio.com.

5. RGPD — protéger les données de tes utilisateurs

Le RGPD est une loi européenne. Elle protège les informations personnelles de tes utilisateurs. Une information personnelle, c'est par exemple : un prénom, un email, une adresse IP. Si tu ne respectes pas le RGPD, tu risques une amende.

La règle de base

Tu ne peux pas collecter des données personnelles sans l'accord de l'utilisateur. Cet accord s'appelle le consentement. Il doit être libre, clair, et l'utilisateur doit pouvoir le retirer à tout moment.

Checklist RGPD — les points essentiels

Avant de mettre ton produit en ligne, vérifie ces 8 points.

  • ✓ Demande l'accord avant de collecter. Si ton site a un formulaire d'inscription, l'utilisateur doit cocher une case pour accepter. Cette case ne doit pas être cochée par défaut.
  • ✓ Explique pourquoi tu collectes. Ton utilisateur doit savoir à quoi servent ses données. Exemple : "Ton email sera utilisé uniquement pour t'envoyer ta confirmation de rendez-vous."
  • ✓ Ne collecte que ce dont tu as besoin. Si tu n'as pas besoin de la date de naissance, ne la demande pas. Moins tu collectes, moins tu risques.
  • ✓ Bandeaux de cookies. Si ton site utilise des cookies (outils de suivi, publicités, analytics), tu dois afficher un bandeau. L'utilisateur doit pouvoir refuser les cookies non essentiels.
  • ✓ Droit de suppression. Ton utilisateur peut demander à supprimer toutes ses données. Tu dois pouvoir répondre à cette demande.
  • ✓ Page de politique de confidentialité. Ton site doit avoir une page qui explique comment tu gères les données. Claude Code peut t'aider à la créer.
  • ✓ Mentions légales. Ton site doit afficher le nom de l'entreprise, l'adresse, et les coordonnées. C'est obligatoire en France.
  • ✓ Pas d'IA générative avec des données sensibles. Ne donne pas à Claude Code des données personnelles réelles (vrais noms, vrais emails). Utilise des données fictives pour tes tests.

Ce que tu peux dire à Claude Code

"Crée une page de politique de confidentialité pour mon site. Mon site s'appelle [nom]. Je collecte : email et prénom. Je n'utilise pas de cookies publicitaires."

6. Accessibilité — un site pour tout le monde

L'accessibilité numérique, c'est s'assurer que ton site fonctionne pour tout le monde. Y compris les personnes malvoyantes, les personnes handicapées, et les personnes âgées. En France, c'est aussi une obligation légale pour de nombreuses organisations.

Ce que ça change concrètement

Un site accessible, c'est un site où les contrastes sont lisibles, les images ont une description, on peut naviguer au clavier, et les formulaires sont bien étiquetés. Claude Code peut faire tout ça si tu lui demandes.

Les règles essentielles à respecter

  • ✓ Contrastes de couleurs suffisants. Le texte doit être lisible sur le fond. Ratio minimum : 4.5:1 pour le texte normal. Tu peux tester sur contrast-ratio.com.
  • ✓ Description des images. Chaque image doit avoir un texte alternatif. Exemple : alt="Photo d'une femme qui sourit en tenant un café". Les personnes malvoyantes lisent ce texte.
  • ✓ Navigation au clavier. Ton site doit fonctionner sans souris. Quelqu'un doit pouvoir naviguer avec la touche Tab.
  • ✓ Titres bien organisés. Chaque page a un seul grand titre (h1). Les sous-titres suivent un ordre logique (h2, h3…). Claude Code gère ça si tu lui demandes.
  • ✓ Formulaires avec étiquettes. Chaque champ de formulaire a un label visible. Exemple : "Ton email :" au-dessus du champ email.
  • ✓ Pas d'animations qui clignotent. Évite les animations rapides. Elles peuvent causer des crises d'épilepsie. Si tu as des animations, elles doivent s'arrêter sur demande.
  • ✓ Texte assez grand. Taille minimale recommandée : 16 pixels pour le texte courant. L'utilisateur doit pouvoir zoomer sans que la mise en page casse.

Ce que tu peux dire à Claude Code

"Vérifie l'accessibilité de cette page. Assure-toi que les contrastes sont suffisants, que les images ont des descriptions, et que la navigation au clavier fonctionne."

Obligation légale en France

Depuis 2023, les entreprises de plus de 250 salariés et les services publics doivent respecter le RGAA (Référentiel Général d'Amélioration de l'Accessibilité). Si tu travailles pour ce type d'organisation, tu dois publier une déclaration d'accessibilité sur ton site.

En résumé — le flux de travail du Batch 3

Voici les étapes dans l'ordre pour créer et publier ton produit.

  1. 1
    Définis ta charte graphique

    Couleurs, police, style. Fournis tout à Claude Code dès le début.

  2. 2
    Crée ton dépôt GitHub

    Un dossier en ligne pour stocker ton code. Gratuit, indispensable.

  3. 3
    Connecte Vercel à GitHub

    Une seule fois. Ensuite, c'est automatique.

  4. 4
    Demande à Claude Code de créer tes pages

    Décris ce que tu veux. Précise la charte graphique, le RGPD, et l'accessibilité.

  5. 5
    Sauvegarde sur GitHub

    Dis à Claude Code : "Sauvegarde sur GitHub, message : [ce que tu as fait]."

  6. 6
    Ton site est en ligne

    Vercel publie automatiquement. Vérifie que tout s'affiche bien sur mobile et ordinateur.

Le message à donner à Claude Code dès le départ

"Je commence un nouveau projet. Il s'appelle [nom]. C'est pour [public cible]. La charte graphique : couleur principale [couleur], police [police]. Respecte l'accessibilité web (WCAG AA) et le RGPD français dans tout le code que tu crées. Mon dépôt GitHub est : [lien]."
